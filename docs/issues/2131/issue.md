# Issue｜Affairs/Build｜Docs｜合并并清理历史 draft/<name> 个人长期分支

## 背景说明

在 FEALPy 的早期研发过程中，曾长期采用以 `draft/<name>` 命名的个人分支进行探索式或阶段性开发。这类分支在当时发挥了重要作用，但随着仓库进入规范化治理与统一研发流程阶段，现有状态存在以下问题：

- 多个 `draft/<name>` 分支长期存在，与当前 `develop` 主研发分支产生结构与历史割裂
- 分支中包含的有效成果未被系统性合并，责任与状态不清晰
- 长期保留个人分支不利于仓库的可维护性、可理解性与协作治理
- 与算海团队当前以 Issue / PR / 证据链为核心的协作契约不一致

在完成 FEALPy 仓库治理与研发前置规范化整理之后，有必要对上述历史个人分支进行一次 **集中、受控、可审计的收口处理**，以消除结构性负担，为后续规范研发扫清分支层面的历史遗留问题。

本 Issue 用于完成该治理动作。

## 目标

完成本 Issue 后，应达到以下可验证目标：

- 所有仍具备价值的 `draft/<name>` 分支内容，已被有序合并至 `develop` 分支
- 合并过程清晰、可回溯，不引入未经确认的技术判断
- 所有历史 `draft/<name>` 个人长期分支已被删除或归档，不再作为研发入口
- FEALPy 仓库仅保留符合当前协作契约的分支结构
- 仓库进入“分支结构治理完成”状态
